This is the layer Orchard manages directly. If you already run a Cashu mint, the
final step is handing it to Orchard so you can tend keys, balances, and mint info
from one dashboard instead of a terminal.

<Tabs>
  <TabItem label="Nutshell">
    [Nutshell](https://github.com/cashubtc/nutshell) is the reference Cashu mint
    implementation — a common starting point and a good match for Orchard to
    manage.
  </TabItem>
  <TabItem label="CDK">
    [CDK](https://github.com/cashubtc/cdk) (the Cashu Development Kit) provides
    `cdk-mintd`, a Rust mint daemon. Point Orchard at a running `cdk-mintd` the
    same way.
  </TabItem>
</Tabs>

## What to have ready

- **A running mint** with its backend reachable from where Orchard runs.
- **The mint's connection details** so Orchard can read its keys, balances, and
  mint info.
- **The Lightning backend** the mint settles through, already connected.

<Aside type="caution" title="Exact connection steps are on the way">
  Orchard is under active development, and the steps for pointing Orchard at
  Nutshell or CDK are still being written — they land here. The
  [Orchard repository](https://github.com/cashubtc/orchard) has the source code in
  the meantime.
</Aside>
