Already running part of the stack? Orchard *manages* a mint rather than replacing
one, so this path is about pointing it at the pieces you already operate. A mint
sits on three layers — Bitcoin, Lightning, and the mint itself. Connect each one
for the service you run, then hand the keys to Orchard.

<Aside type="tip" title="Missing a layer?">
  Each layer leans on the one below it: the mint settles against Lightning, which
  rides on Bitcoin. If you don't have all three yet, build the missing piece with
  the [New Mint](/new-mint/) walkthrough, then come back here to wire it up.
</Aside>
