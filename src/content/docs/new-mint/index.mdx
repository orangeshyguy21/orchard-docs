This is the suggested path to a **fully sovereign mint**: every service run and
hosted on hardware you control. Prepare the server, install and run your services, then
connect Orchard to manage everything from one place.

<CopyForAgent
  block
  label="Copy this guide for your agent"

<Steps>

1. **[System](/new-mint/system/)** — Start by setting up the server you will run your services on.

2. **[Bitcoin node](/new-mint/bitcoin-node/)** — Run your own Bitcoin
  node to act as the source of truth.

3. **[Lightning node](/new-mint/lightning-node/)** — Run a lightning node on
  top of the Bitcoin node for second layer, lightning fast payments.

4. **[Cashu mint](/new-mint/mint/)** — Stand up the mint that uses your
  Lightning node as a backend.

5. **[Orchard](/new-mint/orchard/)** — Install Orchard and point it to
  each service for an all-in-one manager.

</Steps>

Once all five are in place, you can operate the mint from the Orchard dashboard. Already
run some of these services yourself? See [Install](/install/installation/) to connect Orchard to
services you already have.

<Aside type="tip" title="Already have part of the stack?">
  Already running a Bitcoin or Lightning node? Pick up at the step you still
  need. To install Orchard and connect services you already run, see
  [Install](/install/installation/).
</Aside>

## The guides we build on

Rather than reinvent the wheel and maintain our own bare-metal setup steps, we hand the base layers
(System, Bitcoin, and Lightning) off to battle-tested guides that already do this well, and
note the Orchard-specific adjustments on each step. Two guides shaped this path:

  <LinkCard

  <LinkCard

    description="The same best practices, already built for Linux on capable hardware. The purpose-fit path for a public-facing mint, and where our base-layer steps hand off."
  />
</CardGrid>

Orchard's own docs pick up where MiniBolt leaves off, at the Cashu mint and Orchard.

<Aside type="note" title="Node-in-a-box setups">
  Umbrel, Start9, and similar appliances are great for plug-and-play
  self-hosting, but not the right fit here. They run pre-packaged apps with less
  granular control, and are generally geared for private use. A mint is public-facing
  infrastructure so this guide takes a more hands-on approach to setup.
</Aside>
