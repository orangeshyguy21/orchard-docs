AI in Orchard is optional, and off by default. Turn it on and it does two things: it powers
a context-aware **assistant** on the pages themselves, and it adds a background **agent** that
can run scheduled jobs and reach you over Telegram. You switch it on and shape it under
[Settings → Application → AI](/orchard/settings/#ai).

## The integration

Enable AI, then point Orchard at a model provider:

- **Ollama** — local models on your own hardware. Set the **API Endpoint** (it defaults to
  `http://localhost:11434`), and everything stays in-house.
- **OpenRouter** — cloud-hosted models reached through a single **API key**.

**Test connection** confirms Orchard can reach the provider, showing **Connected** or
**Error**. Until you enable AI and set a provider, Orchard runs as a standard application
with the AI features out of the way.

<Aside type="tip" title="Local or cloud">
  Ollama keeps prompts on your own machine; OpenRouter sends them to a third party. If
  keeping your operations private matters, prefer a local model.
</Aside>

## The agent

Enabling AI adds a built-in agent — the **Groundskeeper** — running on your server. An agent
is defined by a **model**, a **description**, a **system prompt** that sets its role and tone,
and a set of **tools**: scoped capabilities across Bitcoin, Lightning, and the mint, shown as
chips that count how many it holds in each group. The tools are what let it actually answer
questions about your stack and act on them, not just chat.

## Messaging (Telegram)

Connect a messaging client and your agent can reach you on your phone. Orchard speaks
**Telegram**: create a bot with [@BotFather](https://t.me/BotFather) using `/newbot`, paste
the token it gives you (Orchard stores it encrypted), and the agent — including scheduled
jobs — can send and answer messages through it. To receive direct messages, add your personal
**Telegram Chat ID** under [Settings → User → Messaging](/orchard/settings/#messaging) — open
a chat with the bot and send `/start` to get it.

## Agent jobs

A **job** pairs an agent with a schedule so it runs unattended and reports back. **New Agent
Job** opens a form: give it a name, pick a model, write a description and system prompt, grant
tools, and set a schedule. Flip it **Active** or **Inactive**, see the next run time, or
**Execute now** to run it on demand — for example, a Lightning Node Watcher that checks node
health every morning and alerts you on failures.

## In-app assistants

Beyond the background agent, Orchard puts a context-aware assistant on the pages
themselves — the **Message assistant…** box at the bottom of every view. Each section has its
own assistant that can read the current view and act on it: adjust the crew table's filters,
fill in mint info or config, narrow the change log, or — as in the screenshot above — build an
agent job for you. When it makes a change, it does so through **tool calls**, and the chat log
shows exactly what it did.
